# TECH_DEBT.md — Руководство по техническому долгу

> Это **руководство по исправлению** проблем по мере их накопления в проекте AstroBot.
> Каждая запись объясняет: где проблема, в чём состоит, почему это плохо, как исправить, как проверить.

---

## Условные обозначения

- 🔴 **Критично** — блокирует масштабирование/разработку, чинить в первую очередь.
- 🟠 **Серьёзно** — мешает развитию, чинить во вторую очередь.
- 🟡 **Средне** — техдолг, чинить когда дойдут руки.

После каждого пункта — чекбокс `[ ]`. Помечайте сделанное.

---

## Текущий долг

На данный момент проект находится на этапе Phase 1 (скелет). Технический долг будет накапливаться по мере реализации. Рекомендуется фиксировать его здесь по мере появления.

### Типовые категории (для будущих записей)

- 🔴 Отсутствие тестов на критическую бизнес-логику (ChatService, SessionLifecycleService)
- 🔴 Хранение ПДн без шифрования
- 🟠 Нет индексов на часто запрашиваемые поля
- 🟠 Путаница с часовыми поясами (naive vs aware datetime)
- 🟡 Магические числа вместо констант
- 🟡 Inline HTML в Python-коде (для Admin API)
- 🟡 Нет CI/CD

Проект следует фазам из `README.md` (Phase 1–5). Техдолг исправляется по мере обнаружения, но до начала следующей фазы все критические и серьёзные проблемы текущей фазы должны быть закрыты.
